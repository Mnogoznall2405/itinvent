# Исправление ошибки с подсказками ФИО при перемещении оборудования

## Проблема
При перемещении оборудования с актом приема-передачи, после ввода команды [/done](file:///C:/Project/Image_scan/bot.py#L275-L275) и ввода фамилии сотрудника, бот вместо показа подсказок ФИО требовал ввод локации.

## Причина
В функции [receive_new_employee](file:///C:/Project/Image_scan/bot.py#L2126-L2182) код был структурирован неправильно:
1. Часть кода, отвечающая за показ подсказок ФИО, была закомментирована или находилась в неправильном месте
2. Отсутствовал обработчик callback для токен-базированного выбора сотрудника в режиме перемещения

## Решение
1. Исправлена функция [receive_new_employee](file:///C:/Project/Image_scan/bot.py#L2126-L2182) - теперь она правильно показывает подсказки ФИО при вводе фамилии сотрудника
2. Добавлен недостающий обработчик callback `handle_select_employee_transfer` для токен-базированного выбора сотрудника

## Изменения в коде
1. В файле `bot.py` исправлена функция [receive_new_employee](file:///C:/Project/Image_scan/bot.py#L2126-L2182) (строки ~2126-2182)
2. В ConversationHandler `transfer_conv` добавлен обработчик callback для паттерна `^transfer:e:`

## Результат
Теперь при перемещении оборудования с актом приема-передачи, после ввода [/done](file:///C:/Project/Image_scan/bot.py#L275-L275) и фамилии сотрудника, бот корректно показывает подсказки ФИО, как и ожидалось.
